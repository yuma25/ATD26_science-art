# 🧠 Technical Knowledge Base (ATD26)

技術的な意思決定とアーキテクチャの根拠を記録します。

---

## 🏗️ アーキテクチャの核心

### データベースと時間の管理 (JST)
- **DB主導の時間管理**: タイムスタンプを日本時間 (JST) で統一。PostgreSQLのデフォルト値やトリガーを利用し、`now() + interval '9 hours'` を設定。これにより、管理画面での直感的な確認が可能になり、アプリケーション側での複雑なタイムゾーン変換を最小化。
- **TypeScript側の責任排除**: データの挿入・更新時にアプリケーション側から現在時刻を送らず、DBのデフォルト値に任せることで、サーバー/クライアント間の時計のズレやUTC/JSTの混在を防止。

### データベースと国際化の簡略化
- **DB主導の名称管理**: 初期設計では多言語対応を見越して `translations.ts` を使用していたが、保守性とシンプルさを優先し、データベースの `name` カラムに直接日本語名を保存する設計に変更。これにより、新しい標本の追加などがDB操作のみで完結する。
- **カラムの最小化**: `description` や `color` などの使用頻度が低く、UI側で個別に定義可能な項目をDBから排除し、スキーマを最小限に保つ。

### BadgeService の環境抽象化
- **環境の自動判別**: `BadgeService` の各メソッドは、ブラウザ側（Client）かサーバー側（Server）かを `typeof window` で判別し、適切な通信手段（fetch API または直接の DB クエリ）を選択する。
- **API経由の書き込み**: データベースの安全性を確保するため、書き込み操作（`acquireBadge`）はクライアントサイドから直接行わず、必ず API ルートを経由してバリデーションと認可を強制する。

### 標本設定の管理 (Specimens)
- **キーの一貫性**: `SPECIMEN_SETTINGS` のキーはデータベースの `name` カラムと完全に一致させる必要がある。現在は「蝶」「クジラ」などの日本語名がキーとして使用されている。

### セキュリティと権限
- **RLS の暫定無効化**: 開発スピードを優先するため `Row Level Security` を無効化しているが、本番運用時はポリシーを適切に設定し、匿名ユーザーが自分のデータのみを操作できるように制限する必要がある。

### Vercel 無料枠での運用最適化
- **リソース制限への対策**: 3DモデルのCDN配信、Redisによる統計データのキャッシュ、SWRによるリクエスト削減を組み合わせることで、無料枠内での安定稼働を実現。
- **3Dモデルの最適化指針**: モバイルAR向けにDraco圧縮やテクスチャサイズ制限（1024px以下）を推奨。目標サイズは1モデル5MB以内。
- **Web Share API による画像保存**: モバイルブラウザの制限（直接ファイル書き込み不可）を回避し、共有シート経由で「写真」アプリへの保存を促す。
avigator.canShare によるファイル共有の可否チェックが重要。

### コンセプト移行と技術的最適化
- **ハイブリッド・レンダリング**: 没入感が必要なARスキャンにはA-Frame(3D)、詳細な鑑賞が必要なビューワーにはNext/Image(2D)を使い分けることで、UXと安定性を両立。
- **SWRによるUX改善**: isLoading（初回）と isValidating（背景同期）を使い分けることで、ユーザーを待たせない「キャッシュファースト」な設計を徹底。
- **モバイル特化UIの工夫**: CloseButton のZ-index最大化や、スマホの縦画面でも横長を維持するチケットUIなど、指の動きと視線誘導を考慮したモバイルファーストな設計。
- **DB整合性の保証**: 外部キー制約違反（23503）に対し、サービスレイヤーで upsert による事前保証を行うことで、匿名認証とデータ保存のレースコンディションを解消。

### 匿名認証の自動化 (useHome / useAR)
- **初回来訪時のセッション確保**: ユーザーがホーム画面に到達した際、useHome フックの useEffect により自動的に signInAnonymously() が実行される。これにより、来場人数入力モーダルなどの「ユーザーIDに紐づく機能」が初回訪問時から即座に利用可能になる。以前は AR 画面への遷移時のみ自動サインインが行われていたが、ホーム画面でも共通化することで UX の一貫性を確保した。

### ARモデルの表示調整方針
- **迫力と視認性の両立**: ユーザーのフィードバックに基づき、デフォルトのスケールを大幅に引き上げ。広大な展示空間ではなく「手元に現れる標本」としての実感を強めるため、Z軸のオフセットを調整し、カメラにより近い位置を移動の基準とした。

### ARモデルの表示調整方針（再）
- **絵画との協調性**: 3Dモデルを巨大化させるのではなく、ARマーカー（絵画）のスケール感に合わせた適切なサイズに縮小。現実の展示物（絵画）から自然に飛び出してきたような一体感を重視した配置（Z軸オフセットの抑制）を採用した。
